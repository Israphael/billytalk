## Цикл 2, веха 2 — что собрано (21.07, на Opus)

Второй процесс интерфейса на wxPython и вся проводка ядро↔интерфейс:
`ui/__main__.py` (коннект через `IpcClient`, чистый выход при потере ядра),
`ui/chrome.py` (4 точки ветвления по ОС из `spec/05`, решаются по номеру билда;
DWM-атрибуты тёмного заголовка и Mica; `MSWEnableDarkMode` в wxWidgets 3.2.9 нет —
перепроверено эмпирически), `ui/overlay.py` — **плашка**: не крадёт фокус
(`SW_SHOWNOACTIVATE`, пять ex-стилей через `SetWindowLongPtrW`), рисуется нами.
Ядро: IPC-сервер (его старт — заодно gate единственного экземпляра),
`state_changed` в канал, ленивый подъём интерфейса на первом PressPTT (OQ §25),
меню трея наполняет интерфейс по IPC (`menu_model`/`menu_command`, OQ §22) со
встроенным минимумом на случай мёртвого интерфейса.

Найден и починен латентный баг из коммита хвостов: `publish` читал
`TrayState.OFFLINE` без импорта — `__main__` не покрыт тестами, проскочило бы до
первого запуска ядра. Добавлен import-smoke обоих `__main__`. Плюс устранены в
корне две тайминг-флаки IPC (`send()` до выставления `_connected`).

⚠ **Визуал плашки (уголки/цвета/шрифт) вживую не проверен** — у сборщика нет
доступа к экрану; Win32-контракт (фокус, ex-стили) закреплён тестами, вид — на
живом прогоне с заказчиком. Живой `python -m billytalk.core` (ядро+интерфейс)
не гонялся: глобальные хуки мыши, нужен присмотр.

## Что нашла реализация (5 багов спеки + 2 раунда ревью)

Проектирование находило ошибки спеки написанием кода со стопроцентной
результативностью. Цикл 1 продолжил: `OPEN-QUESTIONS` вырос с 12 до 20 записей.
Плюс адверсариальное ревью (5 линз, каждая находка перепроверена опровергателем)
дало 11 подтверждённых дефектов в «чистых» модулях — все исправлены с тестами.
Самый серьёзный: `trim_silence` падал на клипе короче одного кадра — ровно на пути
долговечности §3, который редакция 3 переписывалась защищать.

## Что решено и не подлежит переоткрытию

Каждое решение имеет обоснование и раздел «что его отменит» — в `docs/adr/`.

- **Стек: Python 3.14 + wxPython**, ядро отделено от интерфейса. **Не Tauri** — NVDA
  трактует окно WebView2 как веб-страницу и ломает версию 3.0.
- **Пишем с нуля**, заимствуя приёмы с указанием источника. Форк отпал: все кандидаты
  Rust/Tauri, у лучшего нет HTTP-расшифровки вовсе, кнопок мыши нет ни у кого.
- **Хуки остаются в Python** — нативный помощник отменён измерением.
- **Режим 2 стартует прокси в Groq**, аренда GPU отложена до ~5000 часов аудио в месяц.
- **Разбиение MVP-0 / 1.0.** MVP-0 — один режим, ключ заказчика, обслуживает одного
  пользователя.

## Четыре опровергнутых утверждения

Это главное методическое наследие проекта.

| Утверждение | Реальность | Последствие |
|---|---|---|
| «до 1000 колбэков хука в секунду, GIL не выдержит» | **129/с, 0,03% ядра** | отменён целый нативный компонент |
| «WAV быстрее FLAC на 183 мс» (3 прогона) | **FLAC быстрее на 39 мс** (8 пар) | чуть не отменили верное решение |
| «возврат фокуса 70–80%» | **25%** | перевёрнута иерархия доставки |
| **инвариант `count(StartCapture) == count(StopCapture)`** | **невыполним в принципе** — отмена закрывает захват другим эффектом | переформулирован как баланс; найдено только реализацией |

Первые три опровергнуты измерением, четвёртое — написанием кода. **Ни одно
не опровергнуто рассуждением**, и это стоит помнить, прежде чем принимать
следующее правдоподобное утверждение на веру.

**Правило проекта: в API ввода Windows успешный код возврата не доказывает, что
действие состоялось.** Встретилось трижды за два дня в трёх разных API.

## Ориентиры, которые легко потерять

Три жалобы заказчика на Wispr Flow с найденными причинами — это защитные требования,
а не пожелания:

1. **Обрыв диктовки** — гонка на старте, отпускание в состоянии `Initialized`
   трактуется как отмена. → Ставим в очередь. **Второй случай той же болезни найден
   при сборке:** отпускание для диктовки, стоящей в очереди, терялось целиком —
   см. `spec/OPEN-QUESTIONS.md` §9.
2. **Вставка не в то поле** — конкурент сохраняет элемент фокуса и не возвращается
   к нему. → Захват при нажатии, проверка перед вставкой, буфер как основной путь.
3. **История как страховка** — заказчик выживает за счёт неё. → Не второстепенная
   функция.

И принцип, не подлежащий обсуждению: **аудио и текст расшифровки никогда не попадают
в логи**, механизм в `03-harness` §6.

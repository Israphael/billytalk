## Ревью вехи 1 — сделано, но с хвостом

Ultracode-ревью (5 линз × 3 опровергателя) подтвердило и **уже починено**:
смерть потока канала от любого исключения (имя оставалось занятым навсегда —
теперь широкий `except` + `send()` не кидает), клинч на переполнении исходящей
очереди (блокирующий сентинел — теперь только `put_nowait` с разгрузкой),
зомби-соединение (кромочный `CancelIoEx` — теперь уровневое событие смерти
в каждом ожидании), гибель от толстого ответа (>1 МиБ — теперь сообщение
отбрасывается с логом). На каждое — регрессионный тест.

⚠ **Опровергатели семи находок упали на лимите сессии — находки НЕ проверены.
Закрыть первым делом** (быстрая самопроверка или точечный ultracode):

- тултип офлайна без **N** («N записей ждут связи» — спека §3 требует число);
- `IpcClient` не переподключается после `close()` — нужен новый объект (учесть в каркасе UI);
- `HiddenWindow` при `ERROR_CLASS_ALREADY_EXISTS` работает на чужом WNDPROC (тестовый сценарий);
- альфа значка не верифицирована тестом (невидимый значок прошёл бы сьют);
- меню-диспатч (`TrackPopupMenuEx`→`on_command`) без покрытия;
- `FILE_FLAG_FIRST_PIPE_INSTANCE` проверен только со стороны атакующего;
- TOCTOU по PID в проверке образа сервера (теоретический, записать вывод).

Опровергнуто по делу (не трогать): `FlushFileBuffers` после `protocol_mismatch` —
намеренное, спека-обоснованное поведение.

## Измерено циклами 1–2 — не переоткрывай и не забывай

- UIA `TextPattern` — сигнал вставки, ~152 мс; `WM_GETTEXT` мёртв;
  `RESTORE_DELAY_MS=300` подтверждён.
- Фоновая доставка в новый Блокнот: ноль из трёх маршрутов; `_WwG` в чёрном
  списке навсегда. Прогретый пул Groq обязателен (350 против 590 мс).
- `MOUSEINPUT.mouseData` для XBUTTON — значение **без сдвига**.
- **Синхронные Read/Write на одном хэндле канала сериализуются ядром** —
  канал только `FILE_FLAG_OVERLAPPED`, оба конца.
- **`CancelIoEx` кромочный** — смерть соединения сигналится уровневым
  событием во всех ожиданиях, не отменой.
- **Блокирующий сентинел в полную очередь = вечный клинч**; поток канала не
  смеет умирать ни от какого исключения — имя остаётся занятым навсегда
  (`ERROR_PIPE_BUSY` до перезапуска ядра).
- `DisconnectNamedPipe` выбрасывает непрочитанное — `protocol_mismatch`
  доносится `FlushFileBuffers`.
- pywin32 312: `CancelIoEx` нет (ctypes), `TrackPopupMenuEx` без суффикса W;
  **GDI argtypes обязательны как restype** — 64-битный HDC переполняет c_int.
- **GDI-перья пишут нулевую альфу в ARGB DIB** — альфа-проход по пикселям
  руками, иначе значок полностью прозрачен.
- uv-венв: `sys.executable` — трамплин; образ процесса `C:\Python314\python.exe`.
- Бюджет промпта Groq: 420 символов кириллицы / 500 латиницы.

## Как здесь принято работать

- **Измеряй, не оценивай.** Чередованием, минимум восемь пар.
- **Не угадывай молча** — лучшее чтение в код, оба варианта в OPEN-QUESTIONS.
- **Тест, зелёный до починки, ничего не стоит.**
- **Адверсариальное ревью — на вехах**: (каркас UI + плашка) → (verify + окна +
  хоткей). Ключевое слово ultracode. ⚠ Урок вехи 1: ревью съело ~3,1 млн
  токенов сабагентов и упёрлось в лимит сессии на половине опровергателей —
  **закладывай ревью в начало окна лимита, не в конец**, и первой задачей
  новой сессии добей непроверенные хвосты (список выше).
- **Веди документацию параллельно.** Репозиторий публичный и портфолийный.
- **Никогда не логируй** аудио, текст расшифровки, заголовки окон, нажатия.
  По каналу IPC текст — только по `id` (`transcription_ready` не несёт текста).
- **Спрашивай заказчика только там, где нужен его выбор.**

меню трея

Вопрос в OPEN-QUESTIONS при сборке трея: спека §11 называет шесть состояний значка без «офлайн», но §3 требует отдельное состояние «офлайн» — здесь нарисовано семь. Решим при реализации, оба прочтения запишем.

Насколько макет честен относительно wxPython:
  • Придёт от ОС само: вид кнопок, полей, списков и меню; скруглённые углы окон на Win11 и прямые на Win10; системный шрифт (Segoe UI Variable ⇄ Segoe UI).
  • Сделаем через стили окна (ctypes): тёмный заголовок на обеих системах; фон Mica — только на Win11, на Win10 вместо него ровный системный фон, как здесь в режиме «Windows 10».
  • Нарисуем сами, одинаково на обеих ОС: плашка, значок трея, карточки-строки настроек (фон-подложка; контролы внутри остаются системными — доступность для экранных дикторов сохраняется).
  • Упрощение макета: тумблеры показаны в стиле WinUI; в приложении это будут системные флажки либо отрисованный тумблер с полноценной озвучкой — решим при сборке окон.

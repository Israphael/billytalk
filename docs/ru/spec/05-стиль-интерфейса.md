# Стиль интерфейса — утверждён заказчиком

**Дата:** 2026-07-20. Основание: требование заказчика («простой и адаптивный под
Windows 11, выглядит как штатная программа Windows 11; адекватно смотрится и на
Windows 10») и утверждённый им интерактивный макет — `05-макет-интерфейса.html`
(переключатели Win11/Win10 и светлая/тёмная, кликабельные настройки, история,
мастер на 7 шагов, плашка, трей).

## Принцип

**Раскладка одна, оформление отдаёт ОС.** Никакого «второго интерфейса» под
Windows 10: приложение использует нативные контролы, поэтому вид кнопок, полей,
списков, меню, скругления углов и системная тема приходят от самой системы.
Ветвление по ОС — ровно четыре точки.

| Что | Windows 11 | Windows 10 | Кто делает |
|---|---|---|---|
| Углы окон и меню | скруглённые | прямые | ОС, сама |
| Контролы | тема Win11 | тема Win10 | ОС, сама (wx = нативные Win32) |
| Шрифт | Segoe UI Variable | Segoe UI | мы: проба шрифта с откатом |
| Фон окна | Mica (`DWMWA_SYSTEMBACKDROP_TYPE`) | ровный системный | мы: атрибут ставится только на Win11 22H2+ |
| Тёмный заголовок | `DWMWA_USE_IMMERSIVE_DARK_MODE` (20) | тот же атрибут (19 на сборке 1809) | мы: одинаково на обеих |
| Значки-глифы | Segoe Fluent Icons | Segoe MDL2 Assets | мы: откат шрифта, коды почти совпадают |

Определение системы: `build >= 22000` → Windows 11. Смена темы на лету —
`WM_SETTINGCHANGE` / `wx.SystemAppearance`.

## Ограничения (из ADR-0001)

- **Рисованные наборы контролов запрещены** (Fluent-имитации): они уничтожают
  UIA-дерево, а v3.0 — версия для незрячих. Нативный вид обязан оставаться
  нативным.
- Карточки-подложки в настройках — допустимы: рисуется только фон панели,
  контролы внутри остаются системными, доступность сохраняется.
- Тумблеры из макета: системный флажок или отрисованный тумблер с полноценной
  UIA-озвучкой — решить при сборке окон, зафиксировать здесь.

## Измеренные факты под этим решением

- venv: wxPython 4.2.5 = **wxWidgets 3.2.9**; `MSWEnableDarkMode` отсутствует
  (появился в wxWidgets 3.3, которого на PyPI на 2026-07-20 нет). Тёмная тема
  собирается руками: DWM-атрибут заголовка + uxtheme `DarkMode_Explorer` для
  списков/скроллбаров + свои цвета фона панелей. Перепроверить PyPI перед
  циклом 3 — выход wxPython на 3.3 упростит этот раздел.
- Плашка и значок трея — собственная отрисовка, одинаковы на обеих ОС.

## Открытое

- Спека §11 называет **шесть** состояний значка трея без «офлайн», §3 требует
  состояние «офлайн»; в макете нарисовано семь. Решение — при сборке трея,
  оба прочтения занести в OPEN-QUESTIONS.

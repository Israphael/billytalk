# Приёмка MVP-0 (спека §16)

Список из §16 — как есть, плюс **как проверить** и **чем закрыто сейчас**.
Три вида отметки:

- **авто** — закрыто автотестом, гоняется в каждом прогоне (`pytest -q`);
- **живьём** — проверяется руками на установленном билде; тест такое не видит;
- **не проверено** — ждёт живого прогона.

Автотесты доказывают **логику** (машина состояний, ladder вставки, хранилище,
протокол). Они не доказывают, что Windows отдаёт хук, что микрофон открывается
и что текст доехал до чужого окна — это и есть предмет живой приёмки.

Ставить билд: `powershell -ExecutionPolicy Bypass -File packaging\install.ps1`.
Логи (без текста и аудио): `%LOCALAPPDATA%\BillyTalk\logs\core.log`.

## Что осознанно НЕ сделано в MVP-0

- **Подпись кода** — сертификата нет; SmartScreen предупредит один раз
  (`packaging/sign.ps1` готов к сертификату, см. `packaging/README.md`).
- **Локальная расшифровка без интернета** — версия 1.0 (спека §12, шаг 5).
- **Отключение плашки** — §11 держит переключатель серым, пока пользователь ни
  разу не воспользовался Ctrl+Alt+Z: до этого плашка — его единственный канал
  обратной связи. **Правка записей и плеер** — версия 2.0 (§19).
- **OPEN-QUESTIONS §32** (осиротевшая кромка при захвате удерживаемой клавиши)
  — low, отложена осознанно.

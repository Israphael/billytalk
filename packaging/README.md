# BillyTalk — сборка и установка (цикл 3, первый тестовый билд)

Один self-contained бандл PyInstaller `--onedir`, две роли в одном
`BillyTalk.exe` (ядро по умолчанию; `--ui <pipe> <image>` — интерфейс).

## Собрать

```
cd C:\BillyTalk
.venv\Scripts\python.exe -m PyInstaller billytalk.spec --noconfirm
```

Результат — `dist\BillyTalk\BillyTalk.exe` (+ его `_internal\`). Бандл ~82 МБ,
в git не попадает (`build/`, `dist/` в `.gitignore`).

## Установить (currentUser, без прав администратора)

```
powershell -ExecutionPolicy Bypass -File packaging\install.ps1
```

Скрипт:
- копирует бандл в `%LOCALAPPDATA%\Programs\BillyTalk` (без версии в пути, §12);
- прописывает автозапуск **ядра** в `HKCU\...\Run` (§12);
- исключает `BillyTalk.exe` из Windows Error Reporting (§13 — чтобы краш-дамп
  с аудио/расшифровкой/ключом не ушёл в CrashDumps и в Microsoft);
- создаёт ярлык в меню «Пуск».

Удалить: `powershell -ExecutionPolicy Bypass -File packaging\uninstall.ps1`
(история, аудио, логи и ключ Groq не трогаются — чистить вручную).

## Запустить и проверить

Запусти `BillyTalk.exe` (ярлык, или из `%LOCALAPPDATA%\Programs\BillyTalk`, или
он сам стартует при следующем входе в Windows). Ядро уходит **в трей** —
значок микрофона возле часов. Затем: зажать боковую кнопку мыши (Mouse 4),
сказать фразу, отпустить — текст должен появиться в активном поле.

Ключ Groq берётся из диспетчера учётных данных (`BillyTalk/groq-api-key`),
он уже сохранён. Логи (без текста и аудио, §13): `%LOCALAPPDATA%\BillyTalk\logs`.

## Что в этом билде ЕСТЬ и чего пока НЕТ

**Есть:** рабочий frozen-бандл (обе роли), автозапуск, ярлык, WER-исключение,
меню трея, окна настроек/истории, плашка, захват хоткея, Ctrl+Alt+Z/X.
Ключ уже в диспетчере учётных данных, так что первый запуск сразу рабочий.

**Пока нет (следующие шаги цикла 3):**
- **Мастер первого запуска** (§12, 7 шагов: микрофон, язык, хоткей, отвязка
  «Назад», предупреждение про сеть, ключ, проверка). Сейчас настройка — через
  окно «Настройки» и уже сохранённый ключ.
- **Иконка приложения** (.ico) — пока дефолтная от PyInstaller.
- **Локализация en** интерфейса (строки русские).
- **Подпись кода** — Windows SmartScreen предупредит о неизвестном издателе
  при первом запуске (это ожидаемо для неподписанного билда).

## Известные риски первого запуска

- Ядро ставит **глобальные хуки мыши** — это штатно, но первый живой запуск
  стоит делать вручную и под присмотром (как прогон цикла 1).
- Проверка вставки (`verify.py`) генерирует comtypes/UIA-модуль в рантайме; в
  frozen это может не собраться — тогда verify молча деградирует до
  `verify_impossible` (текст всё равно в буфере и истории), диктовка не
  ломается. Если так — увидим в первом же прогоне.

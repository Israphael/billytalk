# BillyTalk — сборка и установка (цикл 3)

Один self-contained бандл PyInstaller `--onedir`, две роли в одном
`BillyTalk.exe` (ядро по умолчанию; `--ui <pipe> <image>` — интерфейс).

## Собрать

```
cd C:\BillyTalk
.venv\Scripts\python.exe packaging\make_icon.py          # только если менялся генератор иконки
.venv\Scripts\python.exe -m PyInstaller billytalk.spec --noconfirm
```

Результат — `dist\BillyTalk\BillyTalk.exe` (+ его `_internal\`). Бандл ~82 МБ,
в git не попадает (`build/`, `dist/` в `.gitignore`). Иконка
`packaging/billytalk.ico` **в git есть** — она генерируется скриптом, и тест
`tests/test_icon.py` сверяет файл с генератором, чтобы они не разъехались.

## Установить (currentUser, без прав администратора)

```
powershell -ExecutionPolicy Bypass -File packaging\install.ps1
```

Скрипт:
- копирует бандл в `%LOCALAPPDATA%\Programs\BillyTalk` (без версии в пути, §12);
- прописывает автозапуск **ядра** в `HKCU\...\Run` и снимает залежавшуюся
  запись `StartupApproved`, которая иначе молча отменила бы автозапуск (§12);
- исключает `BillyTalk.exe` из Windows Error Reporting (§13 — чтобы краш-дамп
  с аудио/расшифровкой/ключом не ушёл в CrashDumps и в Microsoft);
- создаёт ярлык в меню «Пуск» (он же нужен для уведомлений, §11);
- регистрирует запись в «Параметры → Приложения» с командой удаления.

Удалить: «Параметры → Приложения → BillyTalk», либо
`powershell -ExecutionPolicy Bypass -File uninstall.ps1` из папки установки.
История, аудио, логи, настройки и ключ Groq не трогаются — чистить вручную.

## Первый запуск

Запусти `BillyTalk.exe` (ярлык, или из `%LOCALAPPDATA%\Programs\BillyTalk`, или
он сам стартует при следующем входе в Windows). Ядро уходит **в трей** — значок
микрофона возле часов.

На первом запуске само открывается **окно мастера** (§12, 7 шагов): микрофон →
язык → кнопка диктовки → отвязка «Назад» в драйвере мыши → предупреждение про
облако и связь → ключ Groq → живая проверка. Пока мастер не дошёл до «Готово»,
он откроется и в следующий раз. Пройти заново: «Настройки → О программе →
Пройти заново».

Логи (без текста и аудио, §13): `%LOCALAPPDATA%\BillyTalk\logs`.
Ключ Groq: диспетчер учётных данных, `BillyTalk/groq-api-key`.

## Подпись кода

Билд **не подписан** — сертификата нет. Первый запуск даст SmartScreen
«Windows защитила ваш компьютер / неизвестный издатель»: «Подробнее» →
«Выполнить в любом случае». Самоподписанный сертификат тут не помогает
(SmartScreen смотрит на цепочку CA и репутацию файла).

Когда сертификат появится:

```
$env:BILLYTALK_PFX = "C:\path\to\billytalk.pfx"
powershell -ExecutionPolicy Bypass -File packaging\sign.ps1
```

Пароль скрипт спросит сам и не выпустит из процесса: **в аргументах командной
строки пароля быть не должно** — их читает любой процесс того же пользователя,
а `Win32_Process` через WMI и вовсе кто угодно на машине. Вместе с самим `.pfx`
на диске это отдало бы закрытый ключ подписи. Поэтому не `signtool /p`, а
`Set-AuthenticodeSignature` с `SecureString` и без дочернего процесса.
Подпись SHA-256 плюс RFC3161-метка времени (без неё подпись умирает вместе со
сроком сертификата).

## Известные риски первого запуска

- Ядро ставит **глобальные хуки мыши** — это штатно, но первый живой запуск
  стоит делать под присмотром (как прогон цикла 1).
- Проверка вставки (`verify.py`) генерирует comtypes/UIA-модуль в рантайме; в
  frozen это может не собраться — тогда verify молча деградирует до
  `verify_impossible` (текст всё равно в буфере и истории), диктовка не
  ломается. Если так — увидим в первом же прогоне.
- Уведомления идут через значок трея (`Shell_NotifyIcon` + `NIF_INFO`).
  Фокусировка внимания их подавляет — это ожидаемо и по §11 не единственный
  канал: звук и вид значка остаются.
